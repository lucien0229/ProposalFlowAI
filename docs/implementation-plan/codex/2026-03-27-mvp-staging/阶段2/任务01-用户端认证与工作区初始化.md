# 任务01-用户端认证与工作区初始化

## 1. 任务目标

交付 Web 进入路径：Auth、Workspace Setup、Dashboard 基础版与 route guard，让用户能进入产品并开始创建 Opportunity。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜页面清单与页面级需求 v1.0.md`
- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`

## 3. 前置条件

- 阶段 1 的 web session / auth 基线可用
- Web layout 与导航骨架已确认

## 4. 主要工作内容

- 实现 Auth 页面与登录/注册入口
- 实现 Workspace Setup 页面与初始化流程
- 实现 Dashboard 基础页（New Opportunity / Recent Opportunities / Trial 卡片）
- 实现 route guard：未登录不可进入、未 setup 强制进 setup

## 5. 交付物

- Auth 页面
- Workspace Setup 页面
- Dashboard 页面基础版
- route guard 与基础数据读取

## 6. 验收标准

- 用户可以通过登录进入产品
- 首次用户会被正确引导到 Workspace Setup
- setup 完成后进入 Dashboard
- Dashboard 能发起新 Opportunity

## 7. 风险与边界

- 本 task 不要求完成完整 Opportunity 工作流
- Dashboard 先满足启动与回到最近机会的最小能力

## 8. 与上下游任务关系

- 为阶段 2 其余所有 Web 任务提供入口基础
