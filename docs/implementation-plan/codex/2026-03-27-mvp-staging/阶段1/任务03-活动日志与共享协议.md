# 任务03-活动日志与共享协议

## 1. 任务目标

建立共享事件体系、过滤协议、分页协议与统一响应/错误协议，为 Web 埋点与 Admin 聚合查询提供稳定的共享底座。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜共享平台边界说明 v1.0.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端 API 需求说明 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端指标与口径说明 v1.0.md`

## 3. 前置条件

- 核心数据库迁移已建立
- shared-config / shared-schemas 可承载常量与 schema

## 4. 主要工作内容

- 落地 `activity_logs` 模型与写入服务
- 固定最小事件集常量（workspace_created、opportunity_created、proposal_draft_generated 等）
- 收口 Web/Admin 共享过滤字段命名、分页字段命名、错误结构
- 为 admin 聚合查询与 web 页面埋点提供统一协议

## 5. 交付物

- `activity_logs` 写入能力
- shared-config / shared-schemas 中的事件常量、过滤协议、错误协议
- 最小测试夹具或示例事件写入路径

## 6. 验收标准

- Web 主流程关键事件可以稳定写入
- admin 与 web 查询字段命名不漂移
- 错误结构与响应 envelope 可被统一消费

## 7. 风险与边界

- 不要在此 task 内过度建设分析仓或预聚合系统
- event 名称要严格遵守冻结文档，不得自行发明变体

## 8. 与上下游任务关系

- 阶段 2 的关键埋点与阶段 3 的漏斗/列表聚合都依赖本 task
