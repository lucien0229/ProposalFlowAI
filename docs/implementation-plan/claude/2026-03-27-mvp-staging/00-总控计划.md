# 00-总控计划（Claude 设计侧｜MVP → Staging）

## 1. 目标

在不阻塞 Codex 实现推进的前提下，交付可直接对照实现的 Web/Admin 页面骨架与导航结构，并在阶段 2/3 页面实现开始前完成冻结。

## 2. 设计侧冻结项（必须稳定）

- Web：主导航结构、关键页面信息架构、路由骨架
- Admin：主导航结构、Overview / List / Detail / Funnel 页面骨架
- 页面状态位：空态 / 错态 / 计费受限提示位 / 表格与指标卡结构约定

## 3. 交付物口径（对 Codex）

以“可实现、低返工”为第一目标，交付物必须满足：
- `.pen` 资产可被打开并按页面/区域定位
- 页面与路由映射清单完整
- 每个关键页面至少标注：主要区块、关键表单/表格/卡片、状态位占位

详见：[设计交接标准](规范/设计交接标准.md)

## 4. 变更控制

- 冻结前：允许快速迭代
- 冻结后：
  - 不允许改导航层级与路由骨架
  - 允许在不改变信息架构的前提下做局部布局微调

详见：[设计冻结规则](规范/设计冻结规则.md)
