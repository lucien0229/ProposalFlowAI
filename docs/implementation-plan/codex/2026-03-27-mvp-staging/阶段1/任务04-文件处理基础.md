# 任务04-文件处理基础

## 1. 任务目标

建立 PDF 上传、complete、处理、失败重试与状态回写链路，为 Opportunity Lead Intake 提供完整文件处理能力。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`
- `docs/share/ProposalFlow AI｜系统架构与技术方案 v1.0.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`

## 3. 前置条件

- 文件相关表迁移已存在
- worker 骨架已建立
- 对象存储接口边界已明确

## 4. 主要工作内容

- 实现 `file_assets` / `file_processing_jobs` 的最小业务层
- 提供 upload-url / complete / detail / retry API
- 建立 worker 对抽取任务的消费与状态更新
- 让 Web 页面可轮询/读取状态与失败原因

## 5. 交付物

- 文件上传与状态 API
- worker 文件任务处理骨架
- 抽取成功/失败状态回写逻辑

## 6. 验收标准

- 状态严格符合 `uploaded → processing → ready | failed`
- retry 会新增 job 记录，不覆盖历史 job
- 前端能获取 `file_status` 与 `latest_job_status`

## 7. 风险与边界

- 本 task 只关注 PDF 文本抽取，不扩展复杂版面还原
- 本地与 staging 的对象存储实现可以不同，但接口语义必须一致

## 8. 与上下游任务关系

- 阶段 2 的 Lead Intake 页面依赖本 task
