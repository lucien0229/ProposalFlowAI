# 任务06-计费基础

## 1. 任务目标

建立试用与订阅状态模型、checkout/portal/webhook 基线与限制解释逻辑，为用户侧 Billing 页面和 Admin Subscriptions&Trials 提供共享底座。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`
- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`
- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`

## 3. 前置条件

- billing 相关表迁移已存在
- web session 与 workspace context 可用

## 4. 主要工作内容

- 固定 trial / billing 状态模型与内部 source of truth
- 提供 checkout-session / portal-session / webhook 基线接口
- 建立 `is_generation_allowed` / `restriction_reason` 统一计算逻辑
- 建立 webhook 幂等与状态回写策略

## 5. 交付物

- billing 业务层与接口
- webhook 接收与回写逻辑
- 限制矩阵的统一解释函数/服务

## 6. 验收标准

- 能创建 checkout / portal session
- webhook 能幂等处理并更新内部状态
- 受限状态能返回明确的 `restriction_reason`

## 7. 风险与边界

- 本 task 只做 backend/shared foundation，不包含 Billing 页面 UI
- 不直接以前端解释 Stripe 原始对象作为口径来源

## 8. 与上下游任务关系

- 阶段 2 的 `任务08-计费与试用页面.md` 与阶段 3 的 subscriptions 页面都依赖本 task
