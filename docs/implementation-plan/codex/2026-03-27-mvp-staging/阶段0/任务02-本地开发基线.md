# 任务02-本地开发基线

## 1. 任务目标

建立本地开发依赖与启动基线，采用“依赖服务容器化 + 应用本机运行”的模式，让 Web/Admin/API/Worker 后续能在统一依赖上联调。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`
- `docs/share/ProposalFlow AI 开发骨架与目录结构设计说明 v1.0.md`

## 3. 前置条件

- monorepo 骨架已创建
- `apps/*` 与 `packages/*` 的最小目录结构已存在

## 4. 主要工作内容

- 建立 `infra/compose/docker-compose.local.yml`
- 容器化本地依赖：PostgreSQL、Redis、（可选）MinIO
- 约定本地 env 示例文件位置
- 编写本地启动/停止/健康检查脚本入口
- 固定本地推荐启动顺序：依赖 → 迁移 → API → Worker → Web → Admin（按需）

## 5. 交付物

- local compose 文件
- 本地 env 示例与说明
- scripts/dev 下的启动/停止脚本
- 本地健康检查说明（或命令）

## 6. 验收标准

- 一条命令可拉起本地依赖服务
- API/Worker/Web/Admin 可按文档顺序连接依赖运行
- 能清楚区分本地依赖与应用进程职责

## 7. 风险与边界

- 本地不要求 admin 默认启用，但要能按需启动
- 本 task 不负责 staging compose
- 对象存储本地实现可留最小占位，但接口边界要先稳定

## 8. 与上下游任务关系

- 为阶段 1 的迁移、文件处理、billing/webhook、本地联调提供底座
