交付用户侧 Billing / Trial 页面与升级入口，接入 billing foundation，并在 UI 层正确展示状态、限制说明、Checkout 与 Portal 入口。

- `任务06-计费基础.md` 已完成 backend/shared 基线
- web session 与 workspace 上下文可用

## 4. 主要工作内容

- 实现 Billing / Trial 页面
- 展示 `trial_status`、`billing_status`、`plan_type`、`current_period_end`、`is_generation_allowed`、`restriction_reason`
- 接入 Upgrade → Checkout、Manage Billing → Portal
- 与 Dashboard / Proposal Draft / Follow-up 的限制提示保持一致

## 5. 交付物

- Billing / Trial 页面
- 前端对 billing 状态的统一渲染与限制解释组件（如需要）
- checkout / portal 跳转流程

## 6. 验收标准

- 用户能看到当前试用/订阅状态
- 可以进入 Checkout 与 Customer Portal
- 受限状态下能看到明确的限制原因
- 页面口径与 webhook 回写的内部状态一致

## 7. 风险与边界

- 不直接展示或解释 Stripe 原始对象细节作为正式口径
- 不在本 task 内实现 production 账单运营能力

## 8. 与上下游任务关系

- 与 Proposal Draft / Follow-up 的受限状态提示形成统一产品体验
