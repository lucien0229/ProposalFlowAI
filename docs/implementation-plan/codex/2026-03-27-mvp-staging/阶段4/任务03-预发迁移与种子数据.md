# 任务03-预发迁移与种子数据

## 1. 任务目标

标准化 staging 的数据库迁移、初始化与最小 seed 流程，保证可重复执行，并且在迁移前具备最小回滚策略。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`
- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`

## 3. 前置条件

- staging 数据库可访问
- 迁移命令可在容器或独立 job 中运行

## 4. 主要工作内容

- 固定 staging 迁移执行方式（容器内执行或独立 job 执行）
- 固定迁移前检查：确认备份 / PITR 或等价机制
- 提供最小 seed：内部 admin 测试账号、样例 workspace（如需要）

## 5. 交付物

- staging migration 脚本或可执行步骤
- seed 脚本与 staging 环境开关
- 迁移与回滚说明

## 6. 验收标准

- staging 从空库可初始化
- staging 版本升级可迁移
- 迁移失败有明确回滚/恢复路径

## 7. 风险与边界

- seed 数据必须与真实环境隔离，不能污染 production
- 不在本 task 内引入复杂数据生成器

## 8. 与上下游任务关系

- 为 staging smoke tests 提供可测试数据基础
