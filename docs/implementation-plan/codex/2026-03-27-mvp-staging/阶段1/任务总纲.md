## 可并行说明

- task-02 必须先建立迁移框架与核心 schema 基线，再进入 task-01 的 auth/session 业务实现
- task-01 与 task-02 需要紧密协同，但迁移基线以 task-02 为准
- task-03 可在 auth/session 与基础迁移定型后并行
- task-04 / task-05 / task-06 可在核心 DB 与共享协议稳定后并行实现
- 从 task-03 开始冻结共享协议（事件常量、过滤字段、响应/错误结构、限制解释口径）；task-04 / task-05 / task-06 对外协议必须复用该定义，不得各自发明变体
