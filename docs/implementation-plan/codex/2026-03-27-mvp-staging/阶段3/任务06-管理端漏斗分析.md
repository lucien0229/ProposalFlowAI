# 任务06-管理端漏斗分析

## 1. 任务目标

交付 Funnel Analytics 页面与 workspace-level funnel API，识别从 workspace 创建到订阅激活的关键流失位置。

## 2. 参考文档

- `docs/admin/ProposalFlow AI｜运营管理端 PRD v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端页面清单与页面级需求 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端 API 需求说明 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端指标与口径说明 v1.0.md`

## 3. 前置条件

- `activity_logs` 最小事件集已稳定写入
- admin auth 与基础聚合查询已完成

## 4. 主要工作内容

- 实现 `GET /api/v1/admin/funnels/workspace`
- 按冻结节点顺序聚合：workspace_created → opportunity_created → lead_brief_generated → discovery_intelligence_generated → proposal_draft_generated → followup_generated → upgrade_clicked → subscription_activated
- 实现 Funnel Analytics 页面与时间窗口切换
- 明确不混入 `signup_completed`

## 5. 交付物

- funnel aggregate API
- Funnel Analytics 页面
- 节点定义、去重逻辑、转化率测试

## 6. 验收标准

- 漏斗起点固定为 `workspace_created`
- 指标对象明确为 workspace，不混用 user-level 数据
- 能清晰展示主要流失位置与相邻转化率

## 7. 风险与边界

- 首版不实现 user funnel
- 首版不强制实现漏斗节点点击跳到过滤后列表

## 8. 与上下游任务关系

- 完成后阶段 3 可视为 Admin P0 闭环
