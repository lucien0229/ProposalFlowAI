# 任务03-管理端工作区

## 1. 任务目标

交付 Workspaces List 与 Workspace Detail，支持平台视角下按 workspace 查看使用状态、商业状态、成员概况、机会摘要与近期活动时间线。

## 2. 参考文档

- `docs/admin/ProposalFlow AI｜运营管理端页面清单与页面级需求 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端 API 需求说明 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端指标与口径说明 v1.0.md`

## 3. 前置条件

- Admin auth / shell 已可用
- `activity_logs`、workspace、billing 与主流程聚合基础已存在

## 4. 主要工作内容

- 实现 `GET /api/v1/admin/workspaces`
- 实现 `GET /api/v1/admin/workspaces/{workspace_id}`
- 实现 Workspaces List：搜索、状态过滤、分页、排序、快捷标签
- 实现 Workspace Detail：summary、members、usage snapshot、recent opportunities、recent activities

## 5. 交付物

- Workspaces list/detail API
- Workspaces List 页面
- Workspace Detail 页面
- 列表过滤/分页/排序测试与 detail 聚合测试

## 6. 验收标准

- 能稳定筛选与分页 workspace
- `trial_status` / `billing_status` / `plan_type` 过滤准确
- detail 足以支持运营排查与商业判断
- 限制字段与用户侧 Billing 口径一致

## 7. 风险与边界

- 首版活动时间线可折叠在 detail 接口中，不强行拆额外接口
- 不实现高风险运营动作

## 8. 与上下游任务关系

- Overview、Subscriptions 页面可跳转进入 Workspace Detail
