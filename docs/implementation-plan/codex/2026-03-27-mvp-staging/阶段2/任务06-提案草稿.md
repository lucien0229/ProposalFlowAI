# 任务06-提案草稿

## 1. 任务目标

交付 Scope Builder / Proposal Draft 核心页与对应 API，支持生成、章节级重生成、编辑当前态、保存版本、恢复版本、导出，并严格执行 rules 与计费限制矩阵。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`
- `docs/web/ProposalFlow AI｜页面清单与页面级需求 v1.0.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`
- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`

## 3. 前置条件

- Lead Brief 与 Discovery 最小可用
- Templates & Rules / effective rules 可用
- AI orchestration 已可服务 proposal payload

## 4. 主要工作内容

- 实现 proposal draft 的 read / generate / patch / section regenerate / save-version / versions / restore / export
- 实现 Proposal Draft 页面（rules control bar、章节编辑区、风险/限制提示区）
- 生成前校验 Lead Brief 与 Discovery 依赖
- 章节重生成必须有覆盖保护，不静默覆盖人工编辑
- 在受限计费状态下阻断 generate、regenerate、save current、save-version、restore、export

## 5. 交付物

- Proposal Draft API
- Proposal Draft 页面
- 章节级重生成与导出能力
- 相关测试（依赖错误、版本恢复、限制矩阵、覆盖保护）

## 6. 验收标准

- 输出包含 assumptions 与 exclusions
- 可编辑任意章节并执行章节级重生成
- effective rules 与 override active 状态可见
- 依赖不足与计费受限时返回明确阻断说明

## 7. 风险与边界

- MVP 不实现复杂模板推荐
- 导出先支持 text / markdown 等最小格式

## 8. 与上下游任务关系

- Follow-up 依赖 current Proposal Draft
- Billing 页面与 Admin Subscriptions 页面会消费相同限制解释
