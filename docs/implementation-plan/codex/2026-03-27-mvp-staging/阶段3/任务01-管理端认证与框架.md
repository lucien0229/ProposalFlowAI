# 任务01-管理端认证与框架

## 1. 任务目标

交付 Admin 端独立登录入口、layout、导航、route guard 与初始化 shell，确保运营端与客户侧入口、登录态、导航和权限语义彻底分离。

## 2. 参考文档

- `docs/admin/ProposalFlow AI｜运营管理端页面清单与页面级需求 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端 API 需求说明 v1.0.md`
- `docs/share/ProposalFlow AI｜共享平台边界说明 v1.0.md`

## 3. 前置条件

- admin session / internal role guard 已建立
- Admin 线框与导航骨架已确认

## 4. 主要工作内容

- 实现 Admin Auth 页面
- 实现 Admin Layout 与主导航（Overview / Workspaces / Users / Subscriptions / Trials / Funnel Analytics）
- 接入 `/api/v1/admin/auth/me` 与 route guard
- 处理登录失败、无权限、会话失效等状态

## 5. 交付物

- Admin Auth 页面
- Admin shell/layout
- 路由保护逻辑

## 6. 验收标准

- `internal_admin` / `internal_analyst` 可进入系统
- 普通 workspace 用户无法进入 admin
- route guard 与客户侧登录态完全分离

## 7. 风险与边界

- 本 task 不承担业务指标与聚合查询页面实现
- 不复用 web app 的路由树与状态上下文

## 8. 与上下游任务关系

- Admin 其余五个模块的前端实现都依赖本 task
