# 任务04-管理端用户

## 1. 任务目标

交付 Users List（P0），支持平台级用户查询、auth_method 过滤、活跃度查看与 workspace 归属追踪。

## 2. 参考文档

- `docs/admin/ProposalFlow AI｜运营管理端页面清单与页面级需求 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端 API 需求说明 v1.0.md`

## 3. 前置条件

- Admin shell 已可用
- users / auth identities / workspace membership 数据可查询

## 4. 主要工作内容

- 实现 `GET /api/v1/admin/users`
- 实现 Users List：搜索、auth_method、created/active 时间过滤、分页、排序
- 预留 User Detail（P1）定义，但不作为本 task 交付目标

## 5. 交付物

- Users list API
- Users List 页面
- 对应查询与过滤测试

## 6. 验收标准

- 可快速定位用户
- auth_method、last_active_at、workspace_count、primary workspace 信息可读
- 不混入 workspace-level 业务统计口径

## 7. 风险与边界

- 首版不强制实现 User Detail 页面
- 不扩展复杂用户管理能力

## 8. 与上下游任务关系

- 与 Workspaces / Overview 构成平台级查询视角补充
