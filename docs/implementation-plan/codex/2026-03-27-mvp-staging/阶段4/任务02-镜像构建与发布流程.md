# 任务02-镜像构建与发布流程

## 1. 任务目标

建立 `web/admin/api/worker` 的镜像构建与发布流，确保 staging 与未来 production 复用同源 Dockerfile / 构建流程。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`

## 3. 前置条件

- 四个服务的 build 产物可生成
- 目标镜像命名与 tag 规则已确定

## 4. 主要工作内容

- 为 web/admin/api/worker 编写 Dockerfile（建议置于 `infra/docker/`）
- 固定镜像命名与 tag（版本号 + commit SHA）
- 提供构建脚本与发布脚本入口（不必过度自动化）

## 5. 交付物

- 四个 Dockerfile
- build/release 脚本与说明
- 镜像 tag 规范

## 6. 验收标准

- 能构建四个镜像
- 镜像可在 staging compose 中运行
- 版本与 commit 可追踪

## 7. 风险与边界

- 不在本 task 内实现完整 CI/CD 部署流水线
- 不在镜像中保存明文 secrets

## 8. 与上下游任务关系

- 依赖 task-01 的 compose 编排约定
- 为 task-03/04 的 staging 部署与验收提供产物
