# 设计冻结规则（MVP → Staging）

## 1. 冻结的目的

阶段2/3 的实现一旦启动，信息架构与整体布局频繁变化会导致高返工。本规则用于限定“允许改什么、不允许改什么”。

## 2. 冻结范围

冻结后以下内容原则上不允许变更：
- Web/Admin 顶层导航层级与核心入口
- 路由骨架与主要页面集合（新增/删除/改路径）
- 关键页面的信息架构（主要区块的存在与顺序）

允许的变更（不引发返工的大类）：
- 不改变区块语义的局部布局微调（间距、分栏比例、次要区块位置）
- 视觉细节（字体、颜色、按钮样式）
- 文案微调

## 3. 冻结时点

- 目标：在 Codex 开始阶段2/3 页面实现前完成冻结
- 冻结由 Claude 在任务文档中标注 Freeze 版本（见 [设计资产命名与存储规范](设计资产命名与存储规范.md)）

## 4. 冻结后的变更流程（轻量）

当必须改动冻结项时：
1. Claude 在任务04中追加变更说明（原因/影响页面/影响路由）
2. Codex 评估改动对已实现页面的返工范围
3. 双方确认后再改 `.pen` 与路由映射

## 5. 判定标准

如果一个变更会导致：
- 需要改大量路由文件、导航组件、页面壳层结构
- 需要把已有页面的主要区块推翻重排

则它属于“冻结项变更”，必须走流程。
