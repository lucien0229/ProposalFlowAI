# 任务03-线索摘要

## 1. 任务目标

交付 Lead Brief Workspace，支持读取、生成、编辑当前工作态、保存版本、查看历史版本、恢复版本。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜页面清单与页面级需求 v1.0.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`
- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`

## 3. 前置条件

- Opportunity 输入链路已可用
- AI orchestration 基线已建立
- lead_brief current + versions schema 已落地

## 4. 主要工作内容

- 实现 lead brief 的 read / generate / patch / save-version / versions list / version detail / restore
- 实现 Lead Brief 页面双栏布局与字段状态展示
- 支持 current payload 编辑、missing information 高亮、history 预览与 restore
- 阻止无提示覆盖已编辑内容

## 5. 交付物

- Lead Brief API
- Lead Brief 页面
- 相关测试（依赖校验、版本语义、恢复语义）

## 6. 验收标准

- 能从 raw input 生成结构化 Lead Brief
- 可编辑当前工作态并保存
- 可保存版本、查看版本、restore 到当前工作态
- restore 不自动创建新版本

## 7. 风险与边界

- MVP 仅要求整体重生成，不要求字段级重生成
- 页面需明确区分事实、推断、缺失与待审阅信息

## 8. 与上下游任务关系

- Proposal Draft 的正式生成依赖 Lead Brief 可用
