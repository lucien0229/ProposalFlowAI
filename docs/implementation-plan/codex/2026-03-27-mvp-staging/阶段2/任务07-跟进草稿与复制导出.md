# 任务07-跟进草稿与复制导出

## 1. 任务目标

交付 Follow-up Workspace，并补齐 copy / export 收尾闭环，让用户能基于 current Proposal Draft 生成后续沟通草稿。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`
- `docs/web/ProposalFlow AI｜页面清单与页面级需求 v1.0.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`

## 3. 前置条件

- current Proposal Draft 可用
- AI orchestration 已支持 follow-up payload

## 4. 主要工作内容

- 实现 follow-up 的 read / generate / patch / save-version / versions / restore
- 实现 Follow-up 页面：scenario selector、输出区、copy CTA、限制提示
- 处理“页面可进入但无 current Proposal Draft 时不可正式生成”的依赖逻辑
- 保持 current + version + restore 语义一致
- 在受限计费状态下，阻断 Follow-up 的 generate、save current、save-version、restore、export，并展示与 Billing 页一致的限制说明

## 5. 交付物

- Follow-up API
- Follow-up 页面
- copy 能力与相关测试

## 6. 验收标准

- 生成结果至少包含 subject / body / CTA
- 无 Proposal Draft 时页面可进入，但正式生成被阻断
- 可以保存版本并 restore
- 在受限计费状态下，Follow-up 的 generate、save current、save-version、restore、export 被正确阻断并返回明确原因
- copy 行为可用且有可追踪埋点

## 7. 风险与边界

- MVP 不强制冻结显式 tone 切换控件
- 不扩展复杂邮件发送能力

## 8. 与上下游任务关系

- 与 Proposal Draft 一起构成用户侧完整主链路收尾
