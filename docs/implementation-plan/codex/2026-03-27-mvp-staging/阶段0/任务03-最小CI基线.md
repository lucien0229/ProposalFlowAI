# 任务03-最小CI基线

## 1. 任务目标

建立最小 CI 基线，只做质量门禁，不做镜像构建和部署，确保阶段 0 起就有统一的质量检查入口。

## 2. 参考文档

- `docs/share/ProposalFlow AI 开发骨架与目录结构设计说明 v1.0.md`

## 3. 前置条件

- monorepo 工作区已建立
- 各 app/package 已有最小启动与构建入口

## 4. 主要工作内容

- 建立 GitHub Actions workflow
- 固定 CI 检查项：lint / typecheck / unit test / build
- 为 web/admin/api/worker/shared packages 暴露统一 script 入口
- 约定失败输出格式，保证能定位到具体 app 或 package

## 5. 交付物

- `.github/workflows/*` 最小 CI 配置
- 根级或各子项目的检查脚本
- CI 使用说明（简短）

## 6. 验收标准

- push / PR 可触发最小 CI
- lint/typecheck/test/build 能独立报告失败
- 不包含镜像构建、staging 部署、production 发布逻辑

## 7. 风险与边界

- 不要为“未来复杂流水线”过度设计
- API 与前端测试若暂未完整，可先提供最小稳定占位，但命令入口必须真实可执行

## 8. 与上下游任务关系

- 为后续各阶段的增量开发提供基础质量门禁
