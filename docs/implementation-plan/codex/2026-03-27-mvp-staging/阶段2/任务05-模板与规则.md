# 任务05-模板与规则

## 1. 任务目标

交付 Workspace Rules 页面、Opportunity Override 与 Effective Rules 读取能力，建立 Proposal Draft 的规则约束层。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`
- `docs/web/ProposalFlow AI｜页面清单与页面级需求 v1.0.md`
- `docs/web/ProposalFlow AI｜API 设计 v1.0.md`
- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`

## 3. 前置条件

- 模板定义与 rule set 数据模型可用
- shared schema 可承载规则校验

## 4. 主要工作内容

- 实现 `GET /api/v1/templates`
- 实现 workspace rules read / update / validate
- 实现 opportunity override read / update / clear
- 实现 effective rules 读取接口
- 实现 Templates & Rules 页面与 Proposal Draft 上下文中的 override 入口

## 5. 交付物

- Templates & Rules API
- Workspace 基线规则页面
- Opportunity override 交互（在 Proposal Draft 当前上下文中）

## 6. 验收标准

- Workspace 基线规则可保存并影响 Proposal Draft
- Opportunity override 仅影响当前机会，不回写 workspace 基线
- effective rules 可被明确读取和展示
- assumptions / exclusions 保持稳定章节约束

## 7. 风险与边界

- 不提供独立 recommendation API
- override 入口不能脱离 Proposal Draft 当前上下文

## 8. 与上下游任务关系

- Proposal Draft 的生成、章节重生成与风险提示依赖本 task
