# 任务05-预发交付与运行手册

## 1. 任务目标

输出 staging “可交付与可交接”的最终清单：发布前检查、发布步骤、回滚步骤、排障入口与交接说明。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`

## 3. 前置条件

- staging compose、镜像、迁移、smoke tests 已就绪

## 4. 主要工作内容

- 发布前 checklist（版本、配置、依赖健康度、备份）
- 发布步骤（迁移 → api/worker → web/admin → smoke tests）
- 回滚步骤（镜像回退 + DB 恢复策略说明）
- 交接说明（哪些配置在哪里、常见故障怎么查）

## 5. 交付物

- 发布前 checklist
- 发布/回滚 runbook
- 交接清单

## 6. 验收标准

- 团队可按 runbook 独立完成 staging 发布与回滚
- 关键排障入口与日志位置清晰

## 7. 风险与边界

- staging 不等同 production，但流程应尽量同构
- secrets 必须外置管理

## 8. 与上下游任务关系

- 阶段 4 收口任务；完成后 staging 可视为“可验收交付”
