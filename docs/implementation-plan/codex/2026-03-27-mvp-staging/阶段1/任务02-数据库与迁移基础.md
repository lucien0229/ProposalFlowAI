# 任务02-数据库与迁移基础

## 1. 任务目标

按数据库设计文档建立统一迁移体系与核心表结构，固定 current resource + version history 的数据基线。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`
- `docs/share/ProposalFlow AI 开发骨架与目录结构设计说明 v1.0.md`

## 3. 前置条件

- API 工程骨架已存在
- 本地 Postgres 可用

## 4. 主要工作内容

- 建立 Alembic 迁移目录与迁移运行入口
- 按冻结顺序落核心表、外键、唯一约束、索引、check constraints
- 固定 current tables 与 version tables 的关系
- 建立最小 seed 能力（模板、默认枚举、必要测试角色等）

## 5. 交付物

- Alembic 基础配置
- 核心 schema migrations
- seed 脚本与样例数据入口

## 6. 验收标准

- 从空库可执行完整迁移
- 关键约束符合数据库设计文档
- current + version 模型能支撑后续版本保存/恢复语义

## 7. 风险与边界

- 不在本 task 内实现所有业务 service
- admin 报表相关查询先不做预聚合层

## 8. 与上下游任务关系

- 阶段 1 其余任务、阶段 2/3 的资源实现都依赖本 task
