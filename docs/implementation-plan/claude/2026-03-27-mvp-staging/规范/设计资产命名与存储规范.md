# 设计资产命名与存储规范（Pencil）

## 1. 目标

让 Codex 可以稳定找到并对照实现设计资产；让 Claude 可以迭代但不丢失可追溯性。

## 2. 存放位置

- 设计资产目录：`docs/design/`
- 本阶段正式路径：`docs/design/2026-03-27-mvp-staging/`

## 3. 文件命名

- 主文件：`proposalflow-mvp-staging.pen`
- 备选（按端拆分）：
  - `proposalflow-mvp-staging-web.pen`
  - `proposalflow-mvp-staging-admin.pen`

## 4. 页面与节点命名

- 顶层 Frame：`Web` / `Admin`
- Web 页面：`Web / <Route> / <PageName>`
- Admin 页面：`Admin / <Route> / <PageName>`

示例：

- `Web / /opportunities / Opportunity List`
- `Admin / /admin/overview / Overview`

## 5. 版本规则

- 冻结版本用 tag 或在文件名中体现（任选一种，保持一致）：
  - `proposalflow-mvp-staging.pen` 内标注 `Freeze: v1`，或
  - 文件名加后缀：`proposalflow-mvp-staging.freeze-v1.pen`

## 6. 与任务文档的关系

- 每次冻结后，需在 [任务04-设计与界面基础](../阶段0/任务04-设计与界面基础.md) 中写明：
  - 冻结版本标识
  - 覆盖页面清单
  - 路由映射入口
