# 任务01-认证会话与权限基础

## 1. 任务目标

落地共享身份模型、双会话机制与权限分层，确保 web/admin 的登录态、session、guard 与角色语义彻底分离。

## 3. 前置条件

- 阶段 0 完成
- `任务02-数据库与迁移基础.md` 已建立 Alembic/迁移框架与核心 schema 基线
- API app 与 shared packages 已存在

## 4. 主要工作内容

- 落地 `users`、`user_auth_identities`、`workspace_members`、`internal_role_assignments`、`user_sessions`
- 支持 `session_type = web | admin`
- 建立 web auth 基线接口与 admin auth 基线接口（至少 `/api/v1/admin/auth/me`）
- 分离 workspace role guard 与 internal role guard
- 约定 cookie/session 名称与上下文装载方式

## 5. 交付物

- 数据模型与迁移
- web/admin session 读取与 guard 基线
- auth 相关 API 骨架与最小可验证能力

## 6. 验收标准

- web/admin 会话类型可区分
- `owner/member` 不能作为 admin 访问依据
- `internal_admin/internal_analyst` 可访问 admin namespace
- `/api/v1/auth/me` 与 `/api/v1/admin/auth/me` 语义分离

## 7. 风险与边界

- 不在本 task 内完成完整商业逻辑或页面
- 会话安全实现细节需兼顾后续 staging 子域名部署
