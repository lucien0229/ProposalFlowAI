# 任务01-预发基础设施与编排

## 1. 任务目标

交付 staging 环境的基础设施与 compose 编排文件：在一台 VM 上用 docker compose **仅编排 web/admin/api/worker**，并通过环境变量连接独立 staging 数据依赖（Postgres/Redis/对象存储）。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`

## 3. 前置条件

- 四个服务的运行命令与必要端口已明确
- staging 的独立数据库/Redis/对象存储资源已准备（或至少具备占位配置）

## 4. 主要工作内容

- 设计 staging 目录结构：`infra/compose/docker-compose.staging.yml`、`infra/nginx/*`、`infra/deploy/staging/*`
- 编排 web/admin/api/worker 容器（不把 Postgres/Redis/对象存储跑在同一 compose 中）
- 反向代理与域名路由策略（app/admin/api）
- env 注入与 secrets 管理方式说明

## 5. 交付物

- staging compose 文件
- nginx/反代配置（如需要）
- staging env 模板
- staging 部署说明文档（最短可执行步骤）

## 6. 验收标准

- 在 staging VM 上，能通过 compose 拉起 web/admin/api/worker
- 容器能连接到独立 staging Postgres/Redis/对象存储
- web/admin/api 入口路由清晰

## 7. 风险与边界

- 本 task 不负责镜像构建与发布流
- staging 与 production 的差异要最小化（同源 Dockerfile）

## 8. 与上下游任务关系

- 为后续镜像发布、迁移、smoke tests 提供基础
