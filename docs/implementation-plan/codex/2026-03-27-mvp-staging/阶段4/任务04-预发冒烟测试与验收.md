# 任务04-预发冒烟测试与验收

## 1. 任务目标

固化 staging 的 smoke tests 与验收步骤，覆盖：Web 主链路、Admin P0、billing/webhook、file processing、AI 生成基线。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`
- `docs/admin/ProposalFlow AI｜运营管理端 PRD v1.0.md`
- `docs/share/ProposalFlow AI｜部署与环境搭建说明 v1.0.md`

## 3. 前置条件

- staging 已可部署
- 迁移与 seed 已可执行

## 4. 主要工作内容

- 定义 Web 主链路 smoke tests 步骤
- 定义 Admin P0 页面与核心查询 smoke tests 步骤
- 定义 file upload → processing → ready|failed → retry 的验证步骤
- 定义 billing webhook 的验证步骤（test-mode）
- 定义 AI 生成成功/失败的最小验证步骤

## 5. 交付物

- smoke tests checklist（可执行步骤/脚本）
- staging 验收 checklist

## 6. 验收标准

- 任何人按 checklist 能复现验收
- 主链路与关键依赖链路都能在 staging 上验证

## 7. 风险与边界

- smoke tests 优先覆盖“会阻塞试用”的关键路径
- 不在本 task 内建设完整 e2e 自动化体系
