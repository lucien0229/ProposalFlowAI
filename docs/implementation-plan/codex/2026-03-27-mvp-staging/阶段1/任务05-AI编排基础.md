# 任务05-AI编排基础

## 1. 任务目标

建立统一 AI orchestration layer，集中处理模型调用、structured outputs、schema 校验、重试与日志记录，避免各业务模块自行拼装。

## 2. 参考文档

- `docs/share/ProposalFlow AI｜系统架构与技术方案 v1.0.md`
- `docs/share/ProposalFlow AI｜数据库设计 v1.0.md`
- `docs/web/ProposalFlow AI｜PRD v1.0（MVP）.md`

## 3. 前置条件

- `ai_call_logs` 表与相关迁移已存在
- shared-schemas 可承载生成 payload schema

## 4. 主要工作内容

- 建立 AI orchestration service 层
- 定义 request schema / output schema / error normalization 基线
- 为 Lead Brief / Discovery / Proposal / Follow-up 提供统一接入模式
- 记录 `ai_call_logs`（model、latency、tokens、success_status、error）

## 5. 交付物

- AI orchestration 核心服务
- 结构化 schema 校验入口
- `ai_call_logs` 写入逻辑

## 6. 验收标准

- 后续生成类模块不需要自行散落实现模型调用
- 失败、日志、schema 验证路径统一
- 至少一个最小生成链路能验证 orchestration 可用

## 7. 风险与边界

- 不冻结具体模型名，只冻结模型路由与配置方式
- 不在本 task 内实现所有业务页面

## 8. 与上下游任务关系

- 阶段 2 的四类生成能力都依赖本 task
