# 任务04-设计与界面基础（Claude / Pencil）

## 0. 说明

- 本任务负责 Pencil 设计资产与线框交付。
- Codex 以前端实现对照本任务交付的冻结资产。

## 1. 任务目标

由 Claude 指挥 Pencil MCP，产出 Web/Admin 的布局骨架、导航结构与关键页面线框，为 Codex 的前端实现提供稳定结构参照。

## 2. 参考文档

- `docs/web/ProposalFlow AI｜页面清单与页面级需求 v1.0.md`
- `docs/admin/ProposalFlow AI｜运营管理端页面清单与页面级需求 v1.0.md`
- `docs/share/ProposalFlow AI 开发骨架与目录结构设计说明 v1.0.md`

## 3. 前置条件

- monorepo app 边界已固定
- Web 与 Admin 的页面清单、导航与边界已确认

## 4. 主要工作内容

- 设计 Web 全局 layout、主导航、Opportunity 子流程壳层
- 设计 Admin 全局 layout、Overview / List / Detail / Funnel 的基础线框
- 固定页面到路由的映射关系
- 明确错误位、空状态、限制提示位、表格与指标卡的基础结构

## 5. 交付物

- `.pen` 设计文件或等价 Pencil 设计资产
- Web 关键页面线框与布局说明
- Admin 关键页面线框与布局说明
- 页面与路由映射清单

## 6. 验收标准

- Web / Admin 导航结构冻结
- 关键页面骨架能被 Codex 直接对照实现
- 阶段 2 / 3 不再反向改动信息架构与整体布局方向

## 7. 风险与边界

- 以“可实现、低返工”为第一目标，不追求无关高保真细节
- 不在此 task 内要求交付完整视觉系统或复杂组件库

## 8. 与上下游任务关系

- 为阶段 2 Web 页面实现、阶段 3 Admin 页面实现提供结构依据
