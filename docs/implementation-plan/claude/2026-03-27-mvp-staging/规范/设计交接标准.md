# 设计交接标准（Claude → Codex）

## 1. 交接目标

让 Codex 可以在不反复询问的情况下，按设计资产直接落地 Web/Admin 页面骨架与导航。

## 2. 最小交接清单（每次冻结必须具备）

- [ ] `.pen` 设计资产（含 Web/Admin 的导航与关键页面线框）
- [ ] 页面与路由映射清单（包含 Web 与 Admin）
- [ ] 每个关键页面的区块结构说明（至少：Header/Sidebar/Main/Primary actions）
- [ ] 状态位说明：空态 / 错态 / loading / 计费受限提示位
- [ ] 表格与指标卡的字段/结构占位（不要求最终视觉）

## 3. Codex 消费方式（建议）

Codex 在实现任务中：
- 将设计作为“前置条件”引用
- 不在实现任务中重复写设计细节
- 若必须改动 IA/导航/路由骨架，按 [设计冻结规则](设计冻结规则.md) 走变更流程

## 4. 路由映射写法（建议模板）

在任务文档中以表格形式交付：

| App | Route | Page name | Notes |
|---|---|---|---|
| web | /auth | Auth | login/register |
| web | /setup | Workspace Setup | forced until complete |
| admin | /admin/overview | Overview | metrics cards |

## 5. 验收口径

- Codex 能在阶段2/3 的页面任务里按路由表逐页落地壳层
- 不需要 Claude 额外解释才能找到对应线框与状态位位置
